 want this to fit cleanly into existing 
patterns, not fight them.

## What to build

A bill tracking system that integrates into the existing scheduler as a new 
panel. Bills are recurring monthly obligations that need to show up before 
their due date, require manual confirmation of payment, and reset automatically 
each month.

## New file: data/bills.json

Store bill definitions here. Structure:

{
  "bills": [
    {
      "id": "bofa_brina_main",
      "name": "BofA Brina",
      "amount": 137.00,
      "amount_variable": false,
      "due_day": 5,
      "lookahead_days": 7,
      "urgency": "red",
      "category": "credit",
      "notes": "14k limit, near maxed, zero interest until Nov 2026",
      "paid_this_month": false,
      "last_paid_month": null
    }
  ]
}

urgency values: "red" (non-negotiable), "yellow" (important), "gray" (can float)
amount_variable: true means amount shown is an estimate, display with ~ prefix

## New class: src/bill_manager.py

Mirror the DataManager pattern. Responsibilities:
- load_bills() / save_bills() — read/write data/bills.json
- get_due_bills(today) — returns bills whose due_day is within lookahead_days 
  of today, or already past due this month and not marked paid
- mark_paid(bill_id) — sets paid_this_month=True, last_paid_month=YYYY-MM, saves
- reset_month_if_needed() — on new month, set paid_this_month=False for all bills
  (hook this into the existing new-day logic in main_window.py)
- get_week1_bills() — returns all bills with due_day 1-7 as a grouped cluster

## New UI widget: src/ui/bill_block.py

A panel that sits below the task blocks in the main window. Style it to match 
existing TaskBlock conventions — same font, same border style, same color 
variables.

Layout:
- Header: "Bills" label + "X due / Y upcoming" count
- Scrollable list of bill rows, each showing:
  - Colored urgency indicator (red/yellow/gray dot or left border)
  - Bill name
  - Amount (prefixed with ~ if variable)
  - Due date ("due 5th" or "OVERDUE" in red if past due unpaid)
  - Checkbox to mark paid — checking it calls bill_manager.mark_paid()
- Week-1 cluster: if today is between 25th and 5th, show a grouped 
  "WEEK 1 CLUSTER" warning header above those bills with total amount
- Paid bills: dim them and move to bottom, don't hide them entirely until 
  next day's lookahead would no longer show them

## Integrate into existing code

main_window.py:
